# Experiments: RefStyleCap Pipeline

This document describes the experimental setup, training procedures, and evaluation pipeline used in the RefStyleCap framework. It covers:

- BLIP baseline evaluation and fine-tuning  
- Domain adaptation and style transfer using BART and FLAN-T5  
- Caption reference validation using semantic similarity  
- Quantitative and qualitative evaluation, including LLM-as-Judge  

The goal is to enable reproducibility and clarity for downstream users and reviewers.

---

## 1️⃣ Baseline Captioning with BLIP

### Models Used

We use two publicly available BLIP variants from the Hugging Face Model Hub:

- `Salesforce/blip-image-captioning-base`  
  - Vision backbone: ViT-B  
  - Lightweight text decoder (12 layers)  
  - Computationally efficient  
  - Suitable for rapid experimentation  

- `Salesforce/blip-image-captioning-large`  
  - Vision backbone: ViT-L  
  - Deeper text decoder (24 layers)  
  - Higher representational capacity  
  - Requires more GPU memory and compute  

Both models were first evaluated in a **zero-shot** setting to establish baseline captioning performance before any domain adaptation.

---

## 2️⃣ Fine-Tuning BLIP on Domain Data

### Dataset

BLIP models were fine-tuned using a **domain-specific private dataset** consisting of:

- **1,281 image–caption pairs**  
- Each image is paired with a curated caption describing domain-relevant entities, vocabulary, and context  
- Captions were created to reflect visual content and narrative consistency with the Harry Potter universe  

---

### Preprocessing

- Images were resized using the BLIP processor  
- Captions were tokenized using the BLIP tokenizer  
- Captions were:
  - Truncated to a maximum of **30 tokens**
  - Padded to ensure uniform sequence length across each batch  

---

### Training Framework

Training was performed using the Hugging Face `Trainer` API.

**Training Configuration**

| Parameter              | Setting                          |
|------------------------|----------------------------------|
| Training precision     | Mixed precision (fp16)           |
| Loss function          | Autoregressive cross-entropy     |
| Batch size             | 2 samples per device             |
| Gradient accumulation  | 4 steps                          |
| Number of epochs       | 3                                |
| Early stopping         | Validation-based monitoring      |
| Checkpointing          | Final model only                 |

---

### Outputs

After fine-tuning:

- Both **BLIP-base** and **BLIP-large** models were saved  
- Corresponding processors were saved for inference  
- These fine-tuned models serve as the **visual grounding stage** for all downstream experiments  

---

## 3️⃣ Caption Generation (Inference)

During inference:

- Only the BLIP image encoder and text decoder are used  
- The model:
  1. Encodes the input image  
  2. Uses cross-attention to condition the decoder on visual features  
  3. Autoregressively generates tokens until an end-of-sequence token is produced  

The output captions at this stage are:

- Visually grounded  
- Factual  
- Domain-adapted  
- Not yet stylistically aligned with the literary tone  

These captions form the **input** to the style-transfer stage.

---

## 4️⃣ Domain Adaptation & Style Transfer

To align captions with the **literary style of the Harry Potter novels**, we use two sequence-to-sequence models:

### Models Used

- `facebook/bart-base`  
- `google/flan-t5-base`  

These models were selected because:

- **BART** performs strongly in paraphrasing and text rewriting  
- **FLAN-T5** supports instruction-style conditioning for controlled generation  

---

### Input–Output Format

- Input: Caption generated by BLIP  
- Output: Stylistically adapted caption that:
  - Preserves semantic content  
  - Adopts domain-specific vocabulary  
  - Matches narrative tone  

---

### FLAN-T5 Style Conditioning

- Captions are prepended with a style indicator (e.g., `style:`)  
- This enables explicit control over stylistic rewriting  
- Beam search is used during inference to improve fluency and diversity  

---

### BART Two-Stage Adaptation

1. **Stage 1: Domain Adaptation**
   - Trained on the Harry Potter book corpus  
   - Adapts internal representations to domain-specific vocabulary and syntax  

2. **Stage 2: Style-Transfer Fine-Tuning**
   - Trained on paired:
     - Raw captions  
     - Harry Potter–style paraphrases  
   - Enables controlled rewriting while preserving meaning  

---

### Training Data

Style-transfer models were trained using:

- **Base dataset**
  - 1,281 captions  
  - 5,968 input–target style pairs  

- **Extended dataset**
  - 64,766 input–target style pairs  

- **Total style pairs**
  - 70,734 input–target caption pairs  

---

## 5️⃣ Caption Reference Validation (Semantic Similarity)

To evaluate how well captions align with narrative references:

- Each generated caption is compared against:
  - Sentences sampled from the target book corpus  

### Sentence Embedding Models Used

- MiniLM  
- BGE  
- MultiQA  

Each caption and reference sentence is:

1. Independently encoded into a fixed-dimensional embedding  
2. Compared using **cosine similarity**  

This provides a quantitative measure of:

- Semantic alignment  
- Narrative consistency  
- Contextual relevance  

---

## 6️⃣ Automatic Evaluation Metrics

We report the following metrics:

- BLEU-4  
- METEOR  
- ROUGE-L  
- BERTScore  
- Cosine similarity of sentence embeddings  

These metrics assess:

- Lexical overlap  
- Semantic similarity  
- Fluency  
- Faithfulness to content  

---

## 7️⃣ LLM-as-Judge Evaluation

To complement automatic metrics, we use an **LLM-as-Judge** framework based on **Gemini 2.5**.

---

### Evaluation Setup

For each test instance, the judge receives:

- The input image  
- Multiple captions generated at different pipeline stages  

The judge assigns a score from **1 to 5** for each criterion.

---

### Evaluation Criteria

- **Factual Accuracy**  
  Alignment with visual content and domain knowledge  
  Penalizes hallucinations and incorrect details  

- **Fluency & Coherence**  
  Grammar, readability, and logical flow  

- **Depth & Nuance**  
  Descriptive richness and contextual insight  

- **Style Following**  
  Alignment with the Harry Potter literary tone  

---

### Purpose

This evaluation:

- Captures qualitative aspects not reflected by lexical metrics  
- Provides secondary confirmation of automatic results  
- Identifies strengths and weaknesses across models  

Consistency between LLM-as-Judge scores and automatic metrics strengthens confidence in results.

---

## 8️⃣ Model Selection for Final Pipeline

Based on:

- Automatic metrics  
- Semantic similarity scores  
- LLM-as-Judge evaluations  

The **BART-based style-transfer model** was selected as the primary decoder for downstream experiments.

---

## 9️⃣ Reproducibility Notes

- All random seeds are fixed  
- Mixed-precision training (fp16) is used  
- Only final checkpoints are saved  
- Beam search is used for style-transfer inference  
- Identical evaluation sets are used across all models  

---

## 10️⃣ Extending to Other Domains

The RefStyleCap pipeline is domain-agnostic.

To adapt it to a new domain:

1. Replace the movie stills dataset  
2. Replace the domain text corpus  
3. Curate paired raw-caption → styled-caption examples  
4. Fine-tune BLIP + BART/FLAN-T5 using the same pipeline
